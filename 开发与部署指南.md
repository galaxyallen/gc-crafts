# GC CRAFTS — 从开发到部署完整指南

> 本文档记录 GC CRAFTS 官网（Next.js 全栈）从本地开发、后台管理、到 Vercel 生产部署的完整流程、注意事项与排错经验。  
> 生产地址：https://gc-crafts.vercel.app（目标域名：https://www.galaxybz.com）

---

## 一、项目概览

| 项目 | 说明 |
|------|------|
| 类型 | B2B 珠宝展示架品牌官网 + 后台 CMS |
| 前台 | 首页、Displays、Trays、Busts、Watches、OEM、Contact（7 页） |
| 后台 | 产品、询盘、页面内容、材料、工厂数据、站点设置 |
| 技术栈 | Next.js 14 · TypeScript · Prisma · PostgreSQL (Neon) · NextAuth · Vercel Blob · Tailwind |

## 二、开发环境准备

### 2.1 系统要求

- **Node.js 18+**（推荐 20 LTS，需 **64 位**）
- **npm** 或 **pnpm**
- **Git**
- Windows 32 位环境无法运行 Next.js SWC，只能部署到 Vercel 或使用 64 位 Node

### 2.2 克隆与安装

```bash
git clone https://github.com/galaxyallen/gc-crafts.git
cd gc-crafts
npm install
```

### 2.3 配置本地环境变量

复制模板并编辑：

```bash
cp .env.example .env
```

本地 `.env` 最少需要：

```env
DATABASE_URL="postgresql://..."          # 本地或 Neon 开发库
NEXTAUTH_SECRET="随机长字符串"
NEXTAUTH_URL="http://localhost:3000"
ADMIN_EMAIL="admin@gccrafts.com"
ADMIN_PASSWORD="你的密码"
```

生成 `NEXTAUTH_SECRET`：

```bash
openssl rand -base64 32
```

### 2.4 初始化数据库

```bash
npx prisma migrate dev    # 或 npm run db:migrate
npm run db:seed           # 写入管理员、示例产品、页面内容
```

### 2.5 启动开发服务器

```bash
npm run dev
```

| 地址 | 用途 |
|------|------|
| http://localhost:3000 | 前台网站 |
| http://localhost:3000/login | 后台登录 |
| http://localhost:3000/admin | 后台管理（需登录） |

## 四、核心功能说明

### 4.1 数据库模型（Prisma）

主要表：

- `Admin` — 后台管理员
- `Product` — 产品（分类、图片、MOQ、SEO）
- `PageContent` — 页面 CMS 区块（hero、OEM、联系页等）
- `Material` — 首页材料展示卡片
- `FactoryStat` — 工厂/OEM 统计数据
- `Setting` — 站点全局设置（公司名、联系方式等）
- `Inquiry` — 询盘表单

### 4.2 认证（NextAuth）

- 方式：邮箱 + 密码（Credentials）
- 会话：JWT
- 保护路由：`middleware.ts` 拦截 `/admin/*`
- API 鉴权：`lib/api-auth.ts` 使用 `getToken({ req, secret })` 读取 Cookie

**注意：**

- `NEXTAUTH_URL` 必须与浏览器地址栏 **完全一致**（含 `http/https`、是否带 `www`）
- 修改 `NEXTAUTH_SECRET` 或鉴权逻辑后，需 **退出重新登录**
- 所有 API 路由导出 `export const dynamic = "force-dynamic"`，避免 PUT/POST 返回 405

### 4.3 图片上传

| 环境 | 行为 |
|------|------|
| 本地（无 Blob） | 保存到 `public/uploads/` |
| Vercel（已连接 Blob） | 上传到 Vercel Blob |

**Private Blob 存储（当前生产配置）：**

1. Vercel Storage 创建 Blob 时若选 **Private**，必须设置：
   ```env
   BLOB_ACCESS=private
   ```
2. 上传使用 `access: "private"`，返回 URL 为 `/api/media?pathname=...`
3. `app/api/media/route.ts` 代理读取私有图片
4. 前台 `next/image` 对 `/api/media` 路径使用 `unoptimized`

**Public Blob 存储（可选）：**

- 创建时选 Public，无需 `BLOB_ACCESS`
- 图片直接返回 `*.public.blob.vercel-storage.com` URL

**限制：** 单张图片建议 < 4.5 MB（Vercel Serverless 请求体上限）

### 4.4 后台修改与前台同步

前台页面使用 ISR 缓存（`revalidate = 3600`，约 1 小时）。

后台保存成功后，相关 API 会调用 `revalidatePublicSite()` 立即刷新：

- `/api/settings`
- `/api/content/[section]`
- `/api/products`、`/api/products/[id]`
- `/api/materials`、`/api/materials/[id]`、`/api/materials/reorder`
- `/api/factory-stats`
- `/api/upload`

若前台仍显示旧内容：强制刷新（Ctrl+F5）或等待 CDN 边缘缓存过期。

## 五、本地开发注意事项

1. **不要提交 `.env`** — 已在 `.gitignore` 中忽略
2. **图片资源** — 种子数据引用 `public/img/` 下的图片，可按需替换
3. **后台请求** — 使用 `adminFetch()` 确保携带 Session Cookie
4. **数据库变更** — 修改 `schema.prisma` 后执行 `npx prisma migrate dev`，并更新 seed 如需要
5. **参考设计** — `frontend-ref/` 为静态 HTML 参考，不参与构建

## 六、部署到 Vercel（完整流程）

### 6.1 方式 A：GitHub 自动部署（推荐）

```
本地开发 → git commit → git push → Vercel 自动构建部署
```

1. 将代码推送到 GitHub：`github.com/galaxyallen/gc-crafts`
2. 打开 [vercel.com/new](https://vercel.com/new) → Import 仓库
3. Framework 自动识别为 Next.js
4. Build Command：`npm run vercel-build`（已在 `vercel.json` 配置）
5. 配置 Storage 与环境变量（见下文）
6. Deploy

### 6.2 方式 B：Vercel CLI 直接部署（GitHub 推送失败时）

适用于国内网络 `git push` 超时的情况：

```bash
npx vercel login
npx vercel link          # 首次关联项目
npx vercel deploy --prod --yes
```

CLI 会直接上传本地代码到 Vercel，不依赖 GitHub。

### 6.3 创建 Storage

#### PostgreSQL（Neon）

1. Vercel 项目 → **Storage** → **Create Database** → **Postgres**
2. 创建后 → **Connect to Project** → 选择 `gc-crafts`
3. 自动注入 `POSTGRES_URL`、`DATABASE_URL` 等变量

#### Vercel Blob（图片）

1. **Storage** → **Create Database** → **Blob**
2. 选择 **Public** 或 **Private**（当前为 Private，名 `gccrafts`）
3. Blob 页面 → **Projects** → **Connect to Project** → `gc-crafts`
4. 自动注入 `BLOB_STORE_ID`、`BLOB_WEBHOOK_PUBLIC_KEY` 等
5. **必须 Redeploy** 后环境变量才生效

### 6.4 环境变量清单

在 Vercel → **Settings → Environment Variables** 配置：

| 变量 | 必填 | 说明 |
|------|------|------|
| `DATABASE_URL` | ✅ | PostgreSQL 连接串（Connect Neon 后自动注入） |
| `DATABASE_URL_UNPOOLED` | ✅ | 非池化直连（迁移用，Neon 自动注入） |
| `NEXTAUTH_SECRET` | ✅ | 随机密钥，`openssl rand -base64 32` |
| `NEXTAUTH_URL` | ✅ | 生产域名，如 `https://gc-crafts.vercel.app` 或 `https://www.galaxybz.com` |
| `ADMIN_EMAIL` | ✅ | 种子管理员邮箱 |
| `ADMIN_PASSWORD` | ✅ | 种子管理员密码 |
| `BLOB_STORE_ID` | ✅ | Connect Blob 后自动注入 |
| `BLOB_ACCESS` | Private 时必填 | 值为 `private` |
| `BLOB_READ_WRITE_TOKEN` | 可选 | 旧版 Token 方式；OIDC 方式用 `BLOB_STORE_ID` |
| `RESEND_API_KEY` | 可选 | 询盘邮件通知 |
| `NOTIFICATION_EMAIL` | 可选 | 接收询盘通知的邮箱 |

**CLI 添加示例：**

```bash
npx vercel env add BLOB_ACCESS production --value private --yes
```

### 6.5 构建流程说明

`npm run vercel-build` 依次执行：

```
prepare-vercel-env.mjs  →  映射 DATABASE_URL
prisma generate         →  生成 Prisma Client
prisma-migrate.mjs      →  用非池化 URL 执行 migrate deploy
next build              →  构建 Next.js
```

**重要：** Neon 连接池（PgBouncer）无法可靠获取 `pg_advisory_lock`，迁移必须使用 `DATABASE_URL_UNPOOLED` / `POSTGRES_URL_NON_POOLING`。已在 `scripts/prisma-migrate.mjs` 中处理。

### 6.6 初始化生产数据

部署成功后，在本地 `.env` 填入线上 `DATABASE_URL`，执行：

```bash
npm run db:seed
```

或通过 Prisma Studio 手动添加管理员：

```bash
npx prisma studio
```

### 6.7 绑定自定义域名

1. Vercel → **Settings → Domains** → 添加 `www.galaxybz.com` 和 `galaxybz.com`
2. DNS 配置：

| 类型 | 主机 | 值 |
|------|------|-----|
| CNAME | `www` | `cname.vercel-dns.com` |
| A | `@` | `76.76.21.21` |

3. 将 `NEXTAUTH_URL` 改为 `https://www.galaxybz.com`
4. **Redeploy**

`vercel.json` 已配置 `galaxybz.com` → `www.galaxybz.com` 301 重定向。

### 6.8 部署区域

`vercel.json` 设置 `regions: ["hkg1"]`（香港），有利于亚太访问。

## 八、常见问题与排错

### 8.1 图片上传：`Cannot use public access on a private store`

**原因：** Blob 存储为 Private，代码使用了 `access: "public"`。

**解决：**

1. Vercel 添加 `BLOB_ACCESS=private`
2. 确认 Blob 已 Connect to Project
3. Redeploy

### 8.2 后台保存失败 / Failed to save

**可能原因：**

| 现象 | 原因 | 解决 |
|------|------|------|
| HTTP 405 | API 被静态缓存 | 确认 API 路由有 `dynamic = "force-dynamic"` |
| HTTP 401 | Session 过期或鉴权失败 | 退出重新登录；检查 `NEXTAUTH_SECRET` |
| HTTP 503 | 缺少 `NEXTAUTH_SECRET` | 在 Vercel 添加后 Redeploy |

### 8.3 后台改了，前台不更新

**原因：** ISR 缓存未刷新。

**解决：** 确认保存 API 调用了 `revalidatePublicSite()`；用户端 Ctrl+F5 强刷。

### 8.4 构建失败：`P1002 advisory lock timeout`

**原因：** `prisma migrate deploy` 通过 Neon 连接池执行，无法获取 advisory lock。

**解决：** 使用 `scripts/prisma-migrate.mjs`（已集成到 `vercel-build`），确保存在 `DATABASE_URL_UNPOOLED`。

### 8.5 登录失败

- `NEXTAUTH_URL` 与浏览器地址不一致
- 未执行 `db:seed`，数据库无管理员
- 密码与 seed 时 `ADMIN_PASSWORD` 不一致

### 8.6 `git push` 超时（国内网络）

- 使用 VPN / 代理
- 或改用 `npx vercel deploy --prod --yes` 直接部署
- 或使用 GitHub Desktop、Gitee 镜像等

### 8.7 国内访问 Vercel 较慢

- Vercel 服务器在海外，国内直连可能慢或偶发超时
- 绑定自定义域名 + 国内 CDN（需备案）可改善
- 后台管理建议海外网络或稳定代理环境使用

## 十、日常运维

### 更新代码并重新部署

```bash
# 方式 1：Git
git add .
git commit -m "描述本次修改"
git push
# Vercel 自动部署

# 方式 2：CLI（push 失败时）
npx vercel deploy --prod --yes
```

### 修改环境变量后

**必须 Redeploy**，否则运行中的实例仍使用旧变量。

### 数据库迁移（schema 变更后）

```bash
# 本地开发
npx prisma migrate dev --name 描述

# 提交 migrations/ 目录后部署
# 生产构建会自动执行 prisma migrate deploy（通过 prisma-migrate.mjs）
```

### 重置开发数据库

```bash
npm run db:reset    # 清空并重新 migrate + seed（仅开发环境！）
```
